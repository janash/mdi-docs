MDI Standard#

@ : Go to next node#

The engine proceeds to the next node. Typically not supported at the @DEFAULT node.

<@ : Receive current node name#

Data Type: MDI_CHAR
Quantity: MDI_NAME_LENGTH

Engine sends the name of its current node.

@INIT_MC : Go to the INIT_MC node.#

The engine performs any initialization operations that are necessary before a molecular dynamics simulation can be performed, proceeding to the @INIT_MD node.

@INIT_MD : Go to the INIT_MD node.#

Engine proceeds to the INIT_MD node.

@INIT_OPTG : Initialize geometry optimization.#

<CDENSITY : Receive Cartesian coordinates for electronic density#

Data Type: MDI_DOUBLE
Quantity: 3 * NDENSITY

The engine sends the Cartesian coordinates of a set of grid points. This command is intended to be used in conjuction with the <NDENSITY and <DENSITY commands; these three commands enable a driver to acquire the electronic density distribution of an engine in a grid representation. See the <DENSITY command for more details.

>CELL : Send cell vectors to resize simulation cell#

Data Type: MDI_DOUBLE
Quantity: 9

Driver sends cell vectors to engine for resizing its simulation cell.

<CELL : Receive cell vectors#

Data Type: MDI_DOUBLE
Quantity: 9

Engine sends cell vectors to driver.

>CELL_DISPL : Send displacement vector for cell origin adjustment#

Data Type: MDI_DOUBLE
Quantity: 3

Driver sends displacement vector to engine to adjust simulation cell origin.

<KE_NUC : Receive nuclear kinetic energy#

Data Type: MDI_DOUBLE
Quantity: 1

Engine sends total kinetic energy of all nuclei.

<LABELS : Receive atom labels#

Data Type: MDI_CHAR
Quantity: MDI_LABEL_LENGTH * NATOMS

Engine sends a label for each atom, intended for human-readable identification.

>LATTICE : Send lattice point charges#

Data Type: MDI_DOUBLE
Quantity: NLATTICE

Driver assigns a lattice of point charges to the engine.

<LATTICE_FORCES : Receive lattice forces#

Data Type: MDI_DOUBLE
Quantity: 3 * NLATTICE

Engine sends forces on lattice charges to the driver.

<MASSES : Receive atom masses#

Data Type: MDI_DOUBLE
Quantity: NATOMS

Engine sends the mass of each atom to the driver.

>MONKHORST-PACK_NPOINTS : Send Monkhorst-Pack grid points#

Data Type: MDI_INT
Quantity: 3

This command is typically expected for use with plane wave DFT engines. The driver sends the engine the number of k-points to generate on a Monkhorst-Pack grid. The engine then uses the k-points generated on this Monkhorst-Pack grid for all further simulations. The number of k-points to generate along each vector of the Brillouin zone, in ascending order of vector.

>MONKHORST-PACK_SHIFT : Send Monkhorst-Pack grid shift#

<NAME : Receive engine name#

<NATOMS : Receive number of atoms#

<NDENSITY : Receive number of density grid points#

>NLATTICE : Send number of lattice points#

>NPOTENTIAL : Send number of potential grid points#

>POTENTIAL : Send potential grid values#

<PE : Receive total potential energy#

<PE_ELEC : Receive electronic potential energy#

<PE_NUC : Receive nuclear potential energy#

@PRE-FORCES : Go to pre-forces node#

Engine proceeds to the next @PRE-FORCES node. Not valid at the @DEFAULT node.

<SPIN_POLARIZATION : Receive spin polarization setting#

>SPIN_POLARIZATION : Send spin polarization setting#

<STRESS : Receive virial stress tensor#

>STRESS : Send virial stress tensor#

>TOTCHARGE : Send total system charge#

<TOTCHARGE : Receive total system charge#

>VELOCITES : Send atomic velocities#

<VELOCITES : Receive atomic velocities#

<VERSION : Receive MDI Library version#
